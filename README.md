## Goal

Keep Pi mostly bare while adding one practical command:

- `/active-models` — lists models available from providers authenticated in `agent/auth.json`

---

## Active Files

### `agent/settings.json`

Loads only one extension:

```json
{
  "extensions": [
    "./extensions/active-models.ts"
  ]
}
```

### `agent/extensions/active-models.ts`

Registers the `/active-models` command.

What it does:

1. Reads authenticated providers from `agent/auth.json`
2. Queries Pi's model registry (`ctx.modelRegistry.getAll()`)
3. Filters models to authenticated providers only
4. Applies optional CLI-like filters and sorting
5. Prints a table with:
   - provider
   - model
   - intelligence (heuristic score)
   - price (input/output per 1M tokens, from model metadata)
   - context window
   - max output tokens
   - thinking support
   - image support

### `agent/auth.json`

Holds provider credentials (OAuth/API keys). The command uses this file as the source of truth for which providers are active.

### `agent/sessions/`

Session history files generated by Pi.

---

## `/active-models` Usage

### Basic

```text
/active-models
```

### Free-text filter

```text
/active-models github
/active-models codex
```

### Options

- `--provider, -p <name>` (repeatable)
- `--grep, -g <text>`
- `--limit, -n <int>`
- `--sort <provider|model|intel|price|context|max-out>`
- `--desc` / `--asc`
- `--min-intel <0..100>`
- `--max-intel <0..100>`
- `--max-price <usd>` (output per 1M tokens)
- `--min-context <n|nk|nm>`
- `--max-context <n|nk|nm>`
- `--help`

### Examples

```text
/active-models --provider github-copilot
/active-models --provider openrouter --max-price 5 --sort price
/active-models --sort intel --desc --limit 20
```

---

## Notes

- Intelligence is currently heuristic (`≈`) in this extension.
- Prices come from Pi model metadata (`model.cost`) when available.
- Provider detection is based on `agent/auth.json`, not hardcoded provider names.

---

## Security / Hygiene

- `agent/auth.json` is ignored by git and should remain secret.
- Session files may contain sensitive context; treat `agent/sessions/` as private data.
